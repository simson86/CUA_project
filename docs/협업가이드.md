# 협업 가이드 (GitHub · 브랜치 + PR)

이 저장소는 **브랜치 + Pull Request(PR)** 방식으로 협업한다.
`main`은 항상 "돌아가는 안정 상태"로 두고, 각자 자기 브랜치에서 작업한 뒤
PR로 서로 확인하고 병합한다. (실시간 동시편집은 하지 않는다.)

```
main ← 항상 안정 상태 (직접 push 안 함)
 ├─ simson/기능A     ← 각자 자기 브랜치에서 작업
 └─ 동료명/기능B
        ↓ 다 되면
      PR 열기 → 서로 확인 → main에 병합
```

---

## 0. 처음 한 번만 (신규 참여자)

### (소유자) 협업자 추가
1. GitHub 저장소 → **Settings → Collaborators → Add people**
2. 동료의 GitHub 사용자명 입력 → 초대 → 동료가 수락

### (동료) 클론 + 환경 세팅
```bash
git clone https://github.com/simson86/CUA_project.git
cd CUA_project
```
이 저장소는 클론만으로는 안 돌아간다. 3가지를 더 맞춘다:

1. **`.env` 만들기** — `.env`는 커밋되지 않으므로 각자 만든다.
   `.env.example`을 복사해 `GEMINI_API_KEY`를 채운다.
2. **파이썬 패키지** — `py`(Python 3.14) 기준으로 설치:
   `py -m pip install google-genai python-dotenv`
   (⚠ `python`이 아니라 `py`. PATH의 `python`은 msys2라 패키지가 없다.)
3. **post-commit 훅 재생성** — 훅은 `.git/`에 있어 클론에 안 따라온다.
   없으면 progress 저널·리포트 자동생성만 안 될 뿐 치명적이진 않다.
   필요하면 소유자에게 훅 스크립트를 받아 `.git/hooks/post-commit`에 넣는다.

## 1. 매번 반복하는 흐름

### ① 최신 main에서 새 브랜치 만들기
```bash
git checkout main
git pull                      # 동료 변경 먼저 받기
git checkout -b 내이름/작업내용   # 예: simson/bench-force-stop
```
브랜치 이름 규칙: `이름/작업내용` (영문·하이픈 권장).

### ② 작업 → 커밋 → 올리기
```bash
# ... 코드 수정 ...
git add -A
git commit -m "설명적인 커밋 메시지"
git push -u origin 내이름/작업내용   # 첫 push만 -u, 이후엔 git push
```

### ③ PR 열고 확인받기
1. push하면 GitHub가 "Compare & pull request" 버튼을 띄운다 → PR 생성
2. 제목·설명에 **뭘 왜 바꿨는지** 적는다
3. 동료가 **Files changed** 탭에서 확인 / 코멘트
4. 서로 OK면 **Merge pull request** → main에 병합
5. 병합 후 브랜치는 삭제해도 됨 (GitHub의 "Delete branch")

### ④ 병합된 내용 내 로컬에 반영
```bash
git checkout main
git pull                      # 방금 병합된 것 포함 최신으로
```

## 2. 충돌이 났을 때

PR 병합 시 "This branch has conflicts"가 뜨면:
```bash
git checkout 내브랜치
git pull origin main          # main 최신을 내 브랜치로 끌어옴
# 충돌 파일을 편집기로 열어 <<<<<< ====== >>>>>> 표시를 정리
git add -A
git commit
git push
```
그러면 PR의 충돌이 해소된다. 브랜치 방식은 **매번이 아니라 병합 시점에 한 번만** 충돌을 정리하면 된다.

## 3. 이 저장소 특有 주의점

- **auto-commit Stop 훅**: Claude Code로 작업하면 매 턴 자동 커밋된다.
  브랜치 위에서는 그 브랜치로만 커밋되므로 안전하다.
  **`main` 위에서는 자동커밋을 건너뛰도록** 설정돼 있다(main 오염 방지) →
  실제 작업은 항상 자기 브랜치를 체크아웃한 상태에서 한다.
  (이 설정은 `.claude/settings.local.json`으로 각 PC마다 별도. gitignore됨.)
- **공유되지 않는 파일**: `.env`, `.claude/settings.local.json`,
  `예약_*.txt`, `*.pptx`, `*.png`, `git_report.html` 은 gitignore라 push해도 상대에게 안 간다.
  결과물을 공유하려면 `docs/`에 정리해 커밋한다.
- **progress 저널**(`docs/progress/*.md`)은 커밋 메시지로 자동 생성되니
  **설명적인 커밋 메시지**를 쓴다.

## 4. 자주 쓰는 명령 요약

| 상황 | 명령 |
|---|---|
| 새 작업 시작 | `git checkout main && git pull && git checkout -b 이름/작업` |
| 올리기 | `git add -A && git commit -m "..." && git push` |
| 첫 push | `git push -u origin 이름/작업` |
| 남의 병합 받기 | `git checkout main && git pull` |
| 내 브랜치에 main 최신 반영 | `git checkout 내브랜치 && git pull origin main` |
| 현재 브랜치 확인 | `git branch` (앞에 `*` 붙은 게 현재) |
