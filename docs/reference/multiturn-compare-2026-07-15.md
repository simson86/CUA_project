## 결론 — 이번엔 실패가 나왔지만, 수준별 "성공률 우열"은 여전히 안 갈렸다

1. **애매한 작업은 실제로 무너진다.** "제일 인기많은 맨투맨"은 canonical 정답이 없어 **전 수준
   0%**. 사고 수준으로 해결되는 문제가 아니라 **작업 정의 자체가 애매**한 케이스.
2. **진짜 성공률은 세 수준 모두 ~56%로 동률.** minimal→medium(≤medium 한정)에서 **깔끔한
   단조 개선은 없었다.** 대신 **어디서 넘어지는지가 다름**: low 는 치킨(작업2)을 압도(3턴 완주)
   했지만 장바구니(작업3)에서 1회 헛答; minimal/medium 은 반대.
3. **가장 큰 위험은 실패가 아니라 '자신감 있는 오답'(false completion).** low run3 처럼 엉뚱한
   화면에서 done 을 선언하면 파이프라인이 성공으로 오인한다. **사고 수준(≤medium)이 이걸
   막아주진 못했다.**
4. **실패는 비싸다.** 작업1은 15턴을 다 소진해 run 당 **$0.055~0.065** (성공 작업의 5~10배).
   못 풀 작업일수록 max-turns 낭비 비용이 큼 → 조기 포기(early-stop) 로직의 값어치.
